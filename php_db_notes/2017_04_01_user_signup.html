User signup

1st April, 2017

Before we can post, we need to have users. Now, user and session management have layers of complexity (such as having cookies or not). With user identities present in our blog, security issues such as CSRF emerge. Popular frameworks solve these problems, and I will not reinvent the wheel. What I show here is the bare minimum needed to give users their own space.
Let's create a users folder inside public/, and create a form: 
<!-- public/users/signup_form.html -->

<!DOCTYPE html>
<html lang="en">
    <head>
        <meta charset="utf-8">
        <title>Code Blog - Sign Up</title>
    </head>
    <body>
        <h1>Sign up</h1>
        <form action="signup.php" method="POST">
            <table>
                <tr>
                    <td>Username</td>
                    <td><input type="username"></td>
                </tr>
                <tr>
                    <td>Password</td>
                    <td><input type="password" type="password"></td>
                </tr>
            </table>
            <input type="submit">
        </form>
    </body>
</html>
The form above takes us to signup.php: 
<?php

// users/signup.php

require("../db_connect.php");
require("login.php");

$dbh = connect($db_credentials);

$username = $_POST["username"];
$password = $_POST["password"];
$password_hash = password_hash($password, PASSWORD_DEFAULT);

$stmt = $dbh->prepare("INSERT INTO user (username, password_hash)
VALUES (:username, :password_hash)");

$stmt->execute([':username' => $username,
                ':password_hash' => $password_hash]);

login($dbh, $username, $password);

header("Location: ../index.php");
Calling login() from login.php will automatically log in the user after creating the account in the database: 
<?php

// users/login.php

function login($dbh, $username, $password) {
    $check_user_sql = "SELECT password_hash, id FROM user WHERE username = :username";
    $stmt = $dbh->prepare($check_user_sql);
    $stmt->execute([':username' => $username]);
    $user_row = $stmt->fetch();
    $password_hash = $user_row['password_hash'];
    $user_id = $user_row['id'];
    
    if (password_verify($password, $password_hash)) {
        session_start();
        $_SESSION['username'] = $username;
        $_SESSION['user_id'] = $user_id;
    } else {
        die('Could not log in $username. Check the user and password combination. Go back');
    }
}
index.php checks whether a user is logged in (by looking at the $_SESSION['username'] variable): 
<?php

// index.php

session_start();

if (isset($_SESSION['username'])) {
    echo("Welcome back, {$_SESSION['username']}."
       . "(id {$_SESSION['user_id']}) ");
    echo('<a href="users/logout.php">Log out</a>');
} else {
    // display login link
    echo('<a href="users/login_form.html">Please log in</a>');
}
logout.php uses the code from the official documentation: 
<?php

// users/logout.php

session_start();
$_SESSION = array();

if (ini_get("session.use_cookies")) {
    $params = session_get_cookie_params();
    setcookie(session_name(), '', time() - 42000,
              $params["path"], $params["domain"],
              $params["secure"], $params["httponly"]
    );
}

session_destroy();

header("Location: ../index.php");
