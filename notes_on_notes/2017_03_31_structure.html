File structure

31st March, 2017

All my notes are written in HTML. I keep three separate "core" files, index.html, template.html, and TODO.html. The filenames and other code-related items are highlighted with the <code> tag.
index.html lists all the available posts: 
<!-- index.html -->

<!DOCTYPE html>
<html lang="en">
    <head>
        <meta charset="utf-8">
        <meta name="viewport" content="width=device-width, initial-scale=1.0">
        <title>Heitor's Notes on Taking Notes - Home</title>
        <link rel="stylesheet" href="css/style.css">
    </head>
    
    <body>
        <div>
            <a href="/index.html">Back to home</a>
        </div>
        
        <h1>Heitor's Notes on Taking Notes</h1>

        <div>
            <a href="2017_03_31_why.html">Why take notes?</a>
        </div>

        <hr>
        <div>
            <a href="TODO.html">To do</a>
        </div>
    </body>
</html>
template.html is the common HTML for every new post. Either save it as the new post's filename, or paste it into the new post's file. 
<!-- template.html -->

<!DOCTYPE html>
<html lang="en">
    <head>
        <meta charset="utf-8">
        <meta name="viewport" content="width=device-width, initial-scale=1.0">
        <title>Heitor's Notes on Notes - </title>
        <link rel="stylesheet" href="css/style.css">
    </head>
    
    <body>
        <div>
            <a href="index.html">Back to home</a>
        </div>

        <h1>POST TITLE</h1>
        <h3>, 2017</h3>

        <div>
            Content
            <pre>
Code and shell
            </pre>
        </div>
    </body>
</html>
When writing a new post, update the HTML <title> and <h1>POST TITLE</h1>. Monospace content like code and file listings should go inside <pre> tags.
The biggest shortcoming of writing technical posts in plain HTML is the lack of syntax highlighting and need to escape < and > with &lt; and &gt;. If you are writing a lot about HTML, you should consider another method that allows embedding pure HTML.
