2. Tutorial

There are three main parts to this website:

  • On the left, in green, is the Code editor. You will spend most of the time writing code there.
  • On the right, in light yellow, is the Book content. It contains the main text of this book.
  • On the bottom, in dark blue, is the Interpreter. It contains the heart of the embedded Python system.

Code editor

The Code editor is a temporary location where blocks (or just a single line) of code can be written. This website stores the last 50 code blocks that were evaluated in your browser's localStorage.

To reliably save your original work, you will have to copy and paste your code to a more permanent location.

Book Content

The Book content includes several code blocks, which appear in a box with a slightly darker background.

Clicking in these code blocks replaces the content of the Code editor (be careful if you have unfinished work!) and also evaluates (processes) the code.

Python Interpreter

The interpreter runs components developed by the Pyodide project.

It is possible to type code directly in the interpreter. However, it behaves just like a standard operating system terminal, and the keyboard actions are different from the Code editor's.

Greetings

Maintaining a long-standing tradition, we will see how to display a greeting in a few different ways.

The function (roughly, a piece of code that performs an action) called print will display its argument (a value given, or passed, to the function) in the Python interpreter or terminal where the code is run.

Exercise: Click on the left panel and type:

print("Hello, World!")

Then, type the combination Ctrl + Enter (press and hold Ctrl, then press Enter). This will send the current code to the interpreter. Observe that the words Hello, World! appear in the interpreter below.

You may type Ctrl + Enter again and again to repeat the code evaluation.

Now, try typing Alt + Enter. The interpreter will receive the code, and the editor will become empty. This keyboard shortcut is convenient when you know you don't have to keep the current code.

In order to retrieve older code, type Ctrl + Up once or more times. Then, Ctrl + Down will scroll backwards in this history.

Typing Esc will clear the current contents of the Code editor.

Interpreter keyboard shortcuts

Working in the interpreter is also possible, though much less convenient if you are writing multiline code. Typing Enter will make the interpreter evaluate a single line of code.

To process multiple lines, you have to type Shift + Enter to go to the next line.

If you get stuck and wish to start over, type Ctrl + C.

I recommend editing code only in the Code editor. However, it's handy to know how a terminal behaves.

Code blocks in the book's text

Throughout the chapters, there will be blocks of text that explain concepts or defines values. They look like this:

our_planet = "Earth"
our_satellite = "Moon"

print("Hello", our_satellite, "from", our_planet)

Clicking inside this block will replace whatever is in the Code editor with its contents and evaluate it. Be careful if you have unevaluated code in the Code editor!

When you're satisfied trying out the parts of this website, continue on to the next chapter.
