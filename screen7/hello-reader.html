1. Hello, Reader!

Welcome and thank you for joining me in this programming journey! I will cover the basics of computer science, geometry, and algebra with a text-based console and two-dimensional graphics to explain concepts.

The programming language you will be working with is called Scheme. It is a minimalistic language and fundamental computer science concepts can be expressed without complicated syntax.

In this book, there will not be any convoluted code. The code editor is fairly limited, and chapter examples will be written as stand-alone entities that won't require loading code from previous chapters.

After finishing this book you will be able to apply what you have learned to a full-fledged language (or languages) of your choice. Just like spoken languages share elements of grammar, programming languages have many concepts in common with each other.

Hello, World!

The traditional "first program" in introductory books displays the words Hello, World! on the screen.

Scheme allows the programmer to type code and immediately run it, without the intermediate compilation stage, which translates human-readable code into machine language code that your computer can run.

This book includes a Scheme interpreter, which is a piece of software that reads Scheme code and runs it. In Scheme terminology, running code is called evaluating it.

There are several code boxes in lighter yellow in this book. Clicking inside any one of them will copy its contents to the code editor on the top-left corner and evaluate it right away. It saves you from having to type them.

Click on the box below to evaluate the Hello, World! program:

(print "Hello, World!")

You should see the Scheme code and below it, the text Hello, World! inside the dark blue box under the editor. This box is called the console, and it displays values that are the results of your code, and values given to the print function.

A function is a collection of code that performs an action. In print's case, the action is to output a value to the console. Details about functions, and how write your own, will be explained later.

The graphics box

Drawing shapes and plotting data plays a big role in this book. Below is a bit of code that draws a bullseye pattern of circles. Click anywhere inside the code box to run it:

(define (concentric radius)
  (circle 0 0 radius))

(clear)
(for-each concentric (iota 4 40 40))

Each portion of the code block above will be explained in the following chapters. For now, I just wished to show what simple Scheme code looks like.

Can you guess how the code is related to the drawing? concentric draws a circle centered at (0, 0) with the given radius, and the iota function creates 4 values, starting at 40, with an increment of 40 at each step.

With the use of for-each to call the concentric function, four circles with radii of 40, 80, 120, and 160 pixels are drawn.

The default dimensions of the graphics box is 360 pixels by 360 pixels, with -180 <= x < 180 and -180 <= y < 180. Later, we will see how to change these limits.

(clear) is added so that the graphics box is cleared before drawing the picture.

Try writing your own values of iota. Remember, the first number is how many circles to draw, the second number is the initial radius, and the last number is how far apart the remaining circles should be.

Try, for example, (iota 2 100 20) and press Ctrl+Enter or Alt+Enter to redraw. You should see two circles with a large initial radius, and a slightly larger circle.

If you wish to clear the code editor's contents after evaluating your code, press Shift+Ctrl+Enter or Shift+Alt+Enter.

To restore previously evaluated code, press Ctrl+Up or Alt+Up. Unevaluated code will be lost, so be careful! Ctrl+Down or Alt+Down will scroll forwards in this list.

Finally, to clear the contents of the code editor, press Esc. The cleared code may be recovered with Ctrl+Up or Alt+Up.
