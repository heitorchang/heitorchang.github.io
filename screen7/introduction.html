1. Introduction

Welcome and thank you for joining me in a journey where I will cover the basics of programming and elementary algebra, using two-dimensional graphics to explain concepts.

The programming language you will be working with is Scheme. Due to its minimalistic syntax, fundamental computer science concepts can be expressed without syntactical complexity. On the other hand, mainstream languages like Python contain a myriad of special characters and keywords to concisely represent data types and modifiers.

In this book, there will not be any excessively complex code. The code editor is fairly limited, and chapter examples will be written as stand-alone entities that won't require loading code from previous chapters.

As an example, here are a few lines of code that draws a bullseye pattern of circles. Click anywhere inside the code box to run it.

(define (concentric radius)
  (circle 0 0 radius))

(for-each concentric (iota 4 30 40))

After finishing this book you will be able to apply what you have learned to a full-fledged language (or languages) of your choice. Just like spoken languages share elements of grammar, programming languages have many concepts in common with each other.

Click below if you wish to clear the graphics box:

(clear)

Hello, World!

The traditional "first program" in an introductory book is one that displays the words Hello, World! on the screen.

Scheme allows the programmer to type code and immediately run it, without the intermediate compiling stage, which translates human-readable code into machine language code that your computer can execute.

This book includes a Scheme interpreter, which is software that reads Scheme code and executes it. You may enter any Scheme code in the box on the top-left corner of this page. Pressing Ctrl+Enter or Alt+Enter executes the code.

To save you from some typing, there will be several code boxes in lighter yellow across the book. Clicking inside any one of them will copy the code to the editor and execute it right away. The term Schemers use more often is to evaluate the code.

Try clicking on the box below:

(print "Hello, World!")

You should see the Scheme code and below it, the text Hello, World! inside the dark blue box under the editor. This box is called the console, and it displays values that are the results of your code, and values given to print.

print is a function, which is a collection of code that performs an action. In print's case, the action is to output a value to the console.

The value we pass to print is "Hello, World!". Textual values such as this are called strings. They must be enclosed in double quotes ("..."), otherwise, Scheme will think you're referring to names of functions or variables. Note that single quotes won't work. They have a special meaning in Scheme.

Finally, notice that we enclose the entire expression with parentheses (...). Without parentheses, the function will not be called. If you try the previous code without the parentheses, it will still run, and misleadingly, "Hello, World!" will still appear, but only because the string was the final value of our code block.

If you look closely, (print "Hello, World") returns #<no value>, while when the parentheses are left out, it will return the value "Hello, World!" instead. The => characters indicate a return value.

When calling an arithmetic function later on, such as + to add numbers, we'll see that without the parentheses, only the final number is displayed, not the expected result of the calculation.

Writing your own function

Okay, World is a bit too generic. Let's try making Scheme say, "Hello, Alan!" or "Hello, Alice!" instead. The simplest way is merely

(print "Hello, Alan")

But we want to make it a bit simpler to change the name without having to re-type "Hello, " whenever we have a new name.

Creating a function in Scheme looks like this:

(define (function-name argument-1 argument-2 many-more-arguments)
  function-body)

Let's call our function greet. In its body, it will call print:

(define (greet name)
  (print "Hello, " name))

In our function, name is an identifier, whose value is bound when we call the greet function. Try the following:

(greet "Alan")

Then, try changing "Alan" to your own name (or another one if your name is Alan) then press Ctrl+Enter or Alt+Enter to evaluate the new code.

You should see Hello, your-name in the console.

If you wish to clear the editor's contents after evaluating your code, hold Shift in addition to Ctrl or Alt when pressing Enter.

At any time, pressing Ctrl+Up or Alt+Up will restore previously evaluated code. Ctrl+Down or Alt+Down will scroll forwards in this list.

Drawing example

Now that you have seen how a simple function works, let's take another look at the graphics box.

The default setting of the graphics box is a Cartesian plane with the limits -180 <= x <= 180 and -180 <= y <= 180. It looks and behaves like the 2-D plane in math textbooks.

The center, (0, 0) is in the middle of the box, positive x values go towards the right, and positive y values go towards the top.

One of the basic elements of drawing is a line. It is a function that takes 4 arguments (values passed to the function). They are:

(line x1 y1 x2 y2)

where (x1, y1) are the coordinates of one endpoint, and (x2, y2) are the coordinates of the other endpoint.

For example, the function call below will draw a diagonal line in the top-right quadrant:

(line 40 20 150 150)

Feel free to change the coordinates and re-evaluate the code by pressing Ctrl+Enter or Alt+Enter.

When you're done trying out the function, the clear function will wipe the contents of the graphics box.

(clear)

The axes may be redrawn, if you wish:

(axes)

In later chapters, I will explain the components of the following block of code, which draws a pattern of lines:

(define (to-top-edge x)
  (line -180 -180 x 180))
(define (to-right-edge x)
  (line -180 -180 180 x))
(define endpoints (iota 37 -180 10))

(stroke-color "red")
(for-each to-top-edge endpoints)
(stroke-color "blue")
(for-each to-right-edge endpoints)
