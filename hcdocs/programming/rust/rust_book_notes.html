Notes for The Rust Programming Language

4.1 What is Ownership?

A variable of the String type can be mutated: 

let mut s = String::from("Hello");
s.push_str(", World!");
println!("{s}");

Returning the memory used by a variable is done by drop.

Assigning the value of an existing heap-allocated variable to another variable moves it. It's a shallow copy followed by invalidating the first variable. 

let s1 = String::from("Hello");
let s2 = s1;
// s1 is invalidated, s1 was moved into s2

To perform a deep copy, use clone: 

let s1 = String::from("Hello");
let s2 = s1.clone();

Types that implement the Copy trait are trivially copied on the stack (such as integers).

Passing a value to a function will move or copy, just like assignment. Returning values can also transfer ownership.

4.2 References and Borrowing

Here's a calculate_length function that has a reference to an object parameter instead of taking ownership: 

fn main() {
    let s1 = String.from("Hello");
    let len = calculate_length(&s1);
    println!("The length of {s1} is {len}.");
}

fn calculate_length(s: &String) -> usize {
    s.len()
}

&s1 creates a reference that refers to the value of s1 but does not own it. The value is not dropped when the reference is no longer used.

The action of creating a reference is called borrowing.

To modify a borrowed value, a mutable reference is needed: &mut s.

Forbidding more than one mutable reference prevents data races.

The Slice Type

A slice is a kind of reference. It refers to a contiguous sequence of elements in a collection. Being a reference, it does not have ownership.

A string slice is a reference to a part of a String (while &String is a reference to the entire string).

The type that signifies "string slice" is written &str. 

let s = String::from("hello world");

let hello = &s[0..5];  // &s[..5] is the same
let world = &s[6..11];  // since the last byte is included, &s[6..] is the same
let entire_contents = &s[..];
