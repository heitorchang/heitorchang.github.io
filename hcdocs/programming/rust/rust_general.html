Rust

Checking your setup

Follow these instructions to install Rust.

Test the setup with rustc --version.

Local documentation

Run rustup doc

Create a new project with Cargo

Check the Cargo installation with cargo --version.

Run: 

cargo new hello_cargo
cd hello_cargo

Packages of code are called crates.

Building and running a Cargo project

Run cargo build to create an executable. To use optimizations, run cargo build --release. It will create an executable in target/release.

Run cargo run to build and also run the executable.

cargo check makes sure the code compiles, but doesn't produce an executable. The executable will be in target/debug.

Printing a backtrace

Run: 

RUST_BACKTRACE=1 cargo run

RUST_BACKTRACE=full cargo run

Command-line arguments example

use std::env;

fn main() {
    let args: Vec<String> = env::args().collect();

    let num: usize = args[1].parse().expect("could not parse");
    println!("input is {}, double input is {}", args[1], num * 2);
}

Ownership (Rust book, 4.1)

The ownership rules are:

  • each value has an owner
  • there can only be one owner at a time
  • when the owner goes out of scope, the value is dropped

The mechanics of passing a value to a function are similar to those when assigning a value to a variable (passing a variable to a function will move or copy). Returning values can also transfer ownership.

The Copy trait

As a general rule, any group of simple scalar values can implement Copy.

Nothing that requires allocation or some form of resource can implement Copy.
