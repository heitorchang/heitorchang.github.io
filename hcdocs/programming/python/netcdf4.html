netCDF4 package

Source: Unidata GitHub

Groups

netCDF 4 files can have a hierarchical structure of groups, similar to a directory tree.

The topmost group is called the root group. 

import netCDF4 as nc

rootgrp = nc.Dataset("tutorial.nc", "w", format="NETCDF4")

Dimensions

The sizes of all variables are defined in terms of dimensions. Dimensions must be created first. A dimension may be unlimited (pass None or 0 when creating it). 

time = rootgrp.createDimension("time", None)
level = rootgrp.createDimension("level", 0)
lat = rootgrp.createDimension("lat", 73)

print(rootgrp.dimensions)
print(len(lat))

Variables

netCDF variables behave like NumPy multidimensional arrays. A variable may be a scalar (which is uncommon), created without a dimensions tuple. 

times = rootgrp.createVariable("time", "f8", ("time",))
year = rootgrp.createVariable("year", "u2")

All dimensions and variables in a Dataset or Group are stored in a Python dict. 

rootgrp.dimensions
rootgrp.variables
rootgrp.variables.keys()

Attributes

Attributes may be found with rootgrp.ncattrs(). 

rootgrp.ncattrs()
rootgrp.description

Reading and writing a variable's data

The data may be treated as an array of the Variable instance. 

import numpy as np

lats = np.arange(-90, 91, 2.5)
latitudes[:] = lats
print(latitudes[:])

Unlike NumPy arrays, netCDF variables with unlimited dimensions will grow along them if data is assigned outside the currently defined range of indices.
