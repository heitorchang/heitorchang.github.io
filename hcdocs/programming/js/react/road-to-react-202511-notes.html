14.html (React state)

Props are used to pass information down the component hierarchy.

State is used to modify information over time.

useState indicates we wish to have a stateful value that changes over time.

const [current_state, state_updater_function] = useState(initial_state);

useState is a React hook.

15.html (Callback handlers)

Information passed with props only go downwards. To pass information up, we use a callback handler.

16.html

State should always be where all components which depend on it can read (via props) and update (via a callback handler) it.

18.html

React props are just a JavaScript object.

Object destructuring:

const user = {
  firstName: 'Robin',
  lastName: 'Wieruch',
};

const { firstName, lastName } = user;

Transform const Search = (props) => { ... } to const Search = ({ search, onSearch }) => ( ... )

Nested destructuring:

const user = {
  firstName: 'Robin',
  pet: {
    name: 'Trixi',
  },
};

const {
  firstName,
  pet: {
    name,
  },
} = user;

It is not always the best (readable) solution, though.

Rest and spread operators

The rest operator happens on the left side of an assignment (const { id, ...userWithoutAddress } = user), while the spread operator happens on the right side.

Use the spread operator to pass the props object to the next component.
