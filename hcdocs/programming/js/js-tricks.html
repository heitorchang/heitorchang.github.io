Finding the overflowing horizontal element

Seen in Bytes newsletter

Real devs know that centering a div is easy, but what makes even the most senior developers cry - that random overflowing element that makes the whole page scroll horizontally.

How could you find that element? with the POWER OF JAVASCRIPT. Just open up the browser's console and run the following snippet. 

  document.querySelectorAll("*").forEach(el => {
    if(el.offsetWidth > document.documentElement.offsetWidth) {
      console.log(el);
    }
  })

Limiting animation to 60 fps

https://chriscourses.com/blog/standardize-your-javascript-games-framerate-for-different-monitors 

const canvas = document.querySelector('canvas')
const ctx = canvas.getContext('2d')

let msPrev = window.performance.now()

const fps = 60
const msPerFrame = 1000 / fps

// DEBUG: track total number of frames drawn
let frames = 0

function animate() {
  window.requestAnimationFrame(animate)

  const msNow = window.performance.now()
  const msPassed = msNow - msPrev

  if (msPassed < msPerFrame) return

  // insert drawing code here

  const excessTime = msPassed % msPerFrame
  msPrev = msNow - excessTime

  // DEBUG
  frames++
}

// DEBUG: show how many frames were drawn per second
setInterval(() => {
  console.log(frames)
}, 1000)
