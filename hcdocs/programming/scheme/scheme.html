The difference between nested and tail-recursive iteration

The key difference is in the order that the conses are run.

The nested version delays the calls until the end. The first cons that happens is the most deeply nested one (cons 'last-element '()). Since conses attach to the front of the list, the last one stays at the end, then the second-to-last cons runs, and so on.

In the tail-recursive version, each repeated call to g immediately calls cons to build the accum list. Since we can easily only access the car of the input list, the first element goes to the end of accum and we end up with a reversed list.
