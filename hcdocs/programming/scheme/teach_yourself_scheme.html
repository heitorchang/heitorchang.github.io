2 Data Types

2.1 Simple data types

  • Booleans: #t #f (boolean? x) (not #t)
  • Numbers: 42 2+3i 22/7 3.1415 = < >
  • Characters: #\a #\newline #\space #\tab (char? x) (char-downcase #\A) (char-upcase #\a)
  • Symbols: (quote xyz) (symbol? 'xyz)

(define xyz 9)

(set! xyz #\c)

2.2 Compound data types

2.2.1 Strings

Strings are sequences of characters. Don't confuse them with symbols, which are data that have a sequence of characters as their name.

(string #\h #\e #\l #\l #\o)

(string-ref s 0)

(string-append "A " "Man " "A " "Plan")

2.2.2 Vectors

Vectors are sequences but their elements can be anything (even vectors, so that multidimensional vectors can be made).

(vector 1 2 3)

(define v (make-vector 5))

vector-ref and vector-set! access and modify vectors. 

(define v (make-vector 5))
(vector-set! v 2 220)
(vector-ref v 2)

2.2.3 Dotted pairs and lists

A dotted pair is a compound value made by combining any two arbitrary values into an ordered couple. 

(define pair (cons 1 #t))
(car pair)
(cdr pair)

(set-car! pair 101)
(set-cdr! pair #f)

To make lists, it is possible to cons multiple values onto the empty list, (). Note that it not self-evaluating, and should be quoted when using it as a value, '().

The '() or (quote ()) represents both the "empty list" and a "mutable list". It is an error to set a value of an immutable (constant) list such as '(1 2 3) though most implementations don't actually signal an error.
